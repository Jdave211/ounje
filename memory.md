# Ounje Style Memory

This file keeps the iOS app visually consistent as we keep rebuilding.

## Core feel

- Bubbly, guided, animated, and premium.
- Never let the product feel like a cold checklist or enterprise admin panel.
- The app should feel like an agent is actively shaping a meal-prep system with you.

## Visual rules

- Keep the current Ounje palette: dark graphite surfaces with electric green as the lead accent.
- Use rounded cards with soft gradients, subtle strokes, and blurred accent glows.
- Large hero sections are fine, but they should not consume more than roughly half the screen.
- Status bar area should stay protected with a blur shield so content never clashes with system chrome.
- Signed-in screens should inherit the onboarding surfaces instead of reverting to flat utility cards.

## Motion rules

- Prefer spring motion over generic fades.
- Selections should feel like they land, lock in, or pulse into place.
- Background motion should be slow and atmospheric: drifting orbs, glow shifts, small icon rotations.
- Use animation to reward progress, not to distract.

## Interaction rules

- The experience should feel guided, not form-heavy.
- If a choice is preselected, show that intentionally with a small entrance animation.
- Keep one-word and short labels on one line whenever possible.
- Avoid clutter: if a section feels obvious or default, phrase it from the exception case instead.

## Copy rules

- Speak directly to the user: "you", not third-person summary language.
- Keep product language confident and clean.
- Avoid robotic helper text and overexplaining.

## Main app continuity

- Dashboard, Plan, and Profile should all feel like extensions of onboarding.
- Reuse animated hero cards, bubbly stat pills, gradient surfaces, and compact section headers.
- Tab navigation should feel alive and tactile, not flat.
